# Cosmic Registration System (Berlinman Edition)

MVP фестивальной системы: регистрация гостей, sci-fi опрос, расчёт допуска, рекомендация кофе, табло прибывших, режим тамады (планеты + задания).

## Стек

- **Frontend:** Vue 3, Vue Router, Tailwind CSS, PWA (vite-plugin-pwa)
- **Backend:** Supabase (Postgres, Realtime, Storage)
- **Оффлайн:** IndexedDB (idb), очередь синхронизации при появлении сети

## Быстрый старт

1. **Установка зависимостей**

   ```bash
   npm install
   ```

   Если возникает конфликт peer dependencies (Vite 7 и vite-plugin-pwa):

   ```bash
   npm install --legacy-peer-deps
   ```

   Либо временно удалите `vite-plugin-pwa` из `package.json` и из `vite.config.js` (плагин PWA), затем снова `npm install`.

2. **Переменные окружения**

   Скопируйте `.env.example` в `.env` и укажите ключи Supabase:

   ```
   VITE_SUPABASE_URL=https://ваш-проект.supabase.co
   VITE_SUPABASE_ANON_KEY=ваш-anon-key
   ```

   Без `.env` приложение работает в оффлайн-режиме (данные в IndexedDB).

3. **Supabase**

   - Создайте проект на [supabase.com](https://supabase.com).
   - В SQL Editor выполните скрипт из `supabase/schema.sql` (таблицы + сид кофе). Для опроса подбора напитков — `supabase/seed_quiz.sql` (примеры вопросов и ответов), обычные космические вопросы — `supabase/seed_quiz_ordinary.sql` и `supabase/seed_quiz_ordinary_2.sql` (ещё 15), рубрика «Смотрящий по галактике спрашивает» — `supabase/seed_quiz_galaxy_watcher.sql`. Для списка аллергенов в опроснике — `supabase/seed_allergens.sql`.
   - В настройках проекта включите Realtime для таблиц `guests` и `coffee_orders` (в schema это уже указано через `alter publication`).

4. **Запуск**

   ```bash
   npm run dev
   ```

   Сборка: `npm run build`. Превью продакшена: `npm run preview`.

## Маршруты

| Путь | Назначение |
|------|------------|
| `/register` | Регистрация: позывной → фото → 4 калибровочных блока → карточка + кофе |
| `/guest/:callsign` | Профиль гостя и рекомендации кофе |
| `/board` | Табло прибывших (последние 20, realtime); для готовых заказов — «Кофе готов, подходи» |
| `/barista` | Табло бариста: очередь заказов, кнопка «Готово» |
| `/admin` | Штаб заданий (центр управления заданиями): круги рас, объединение, таймер, фото |
| `/admin/coffee` | Кофейная карта: напитки с 4 параметрами (сладость, горечь, интенсивность, экстрим), состав, вкл/выкл |
| `/admin/quiz` | Вопросы подбора: добавление вопросов и ответов с дельтами по 4 параметрам, вес вопроса, вкл/выкл |

## Ритуал регистрации

1. **Скан имени** — ввод позывного, проверка на существование.
2. **Сенсор** — «тепловая карта лица» (фото с камеры или выбор файла).
3. **4 калибровочных блока** — толерантность к среде, энергетический уровень, готовность к модификациям, терморежим (без цифр, только варианты).
4. **Результат** — матрица (Инфернис/Лириан/Вулканид/Кристалл/Нейтрал), уровень допуска 1–10, три рекомендованных кофе.

Расчёт: `total_score = entropy + resonance + logic`, `clearance_level = round(total_score/3)` в границах 1–10.

**Подбор напитков по 4 параметрам:** после калибровки гостю показываются случайные активные вопросы из БД (по умолчанию 5). Каждый ответ даёт дельты по осям: сладость, горечь, интенсивность, экстрим. По сумме (с учётом веса вопроса) строится профиль 0–10, затем по евклидовой дистанции выбираются ближайшие напитки. Учитывается фильтр по аллергиям (исключаются напитки с запрещёнными ингредиентами). Если в БД нет вопросов — используется прежняя логика по `clearance_level` и `experimental_level`.

## Оффлайн

- Регистрация сохраняется в IndexedDB.
- Фото хранится в данных гостя (data URL).
- При событии `online` выполняется `flushPendingSync()` — отправка накопленных гостей в Supabase (upsert по `id`).
