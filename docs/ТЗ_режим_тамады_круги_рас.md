# ТЗ: Штаб заданий (центр управления заданиями) — круги рас, авто-запуск, таймер, фото

## 1. Общая идея

Режим «Штаб заданий» (центр управления заданиями) показывает **все расы** в виде кружков (планет), расположенных **по кругу**. В каждый кружок попадают участники этой расы (гости из очереди на кофе с соответствующей расой по квизу). Задание для расы **запускается автоматически**, когда в кружок набирается заданное число человек (настраивается слайдером 3–10). Во время выполнения задания показывается крупный текст задания и обратный отсчёт 5 минут. Задание считается выполненным после **прикрепления фото** к планете; съёмка фото сопровождается таймером 3 секунды. Если во время выполнения задания в ту же расу «попал» ещё один человек (заполнил анкету и оказался в этой расе) и набралось новое пороговое количество — создаётся **новая группа (планета)** для них; старая группа продолжает выполнять своё задание.

---

## 2. Экран: круги рас

- **Расы** берутся из БД (`races`), отображаются как **кружки (планеты)**, расположенные **по кругу** (равномерно по окружности).
- Внутри каждого кружка:
  - **Участники этой расы** — гости, которые в очереди на кофе (заказ в статусе `pending` или `ready`; **не** `picked_up`), у которых раса = эта. Даже когда кофе готов (`ready`), человек остаётся в кружке — не исчезает. Исчезает из кружка только когда попал в планету и **планета завершена** (приложено фото). То есть для подсчёта «в кружке»: заказ есть (pending/ready), раса совпадает, гость **не** входит в состав активной планеты.
  - На каждого участника показывается **аватарка**: если есть `avatar_url` — фото, иначе — **аватарка с вопросиком** (placeholder).
  - **Счётчик**: сколько всего участников в этом кружке (например: «3 / 5» или «5 участников»).
- Общее число участников по всем расам (или общее по экрану) должно быть **очевидно видно** (например, заголовок «Всего участников: N» или сумма по кружкам).

---

## 3. Настройки штаба заданий

- **Слайдер «Запуск задания при N участниках»**: минимум 3, максимум 10. Значение сохраняется (localStorage и/или позже — настройка в БД/админке).
- Когда в кружок расы набирается **ровно или больше** этого N — для этой расы **автоматически** создаётся планета и запускается задание (без ручной кнопки «Объединить»). Ручной запуск по желанию можно оставить как опцию.

---

## 4. Запуск задания

- В момент достижения порога (N человек в кружке расы):
  - Создаётся **планета** (группа) для этих N человек этой расы, им выдаётся задание (случайное из `tamada_tasks` или выбранное — по текущей логике).
  - Эти участники **уходят из «очереди в кружке»** в состояние «выполняют задание» (они уже в `planet_members` активной планеты).
- На экране:
  - Для этой расы (или в отдельном блоке «Текущее задание») показывается **текст задания очень крупным шрифтом**.
  - Запускается **обратный отсчёт 5 минут** (часы:минуты:секунды или минуты:секунды) для этой планеты.

---

## 5. Таймер планеты

- У активной планеты — **таймер 5 минут**. По истечении планета **не исчезает**: задание помечается как **«просрочено»** (визуально: надпись/стиль), но фото по-прежнему можно приложить и завершить задание. Планета исчезает только после прикрепления фото (статус `completed`).

---

## 6. Новая группа, если во время задания пришли ещё в ту же расу

- Пока планета **выполняет задание** (статус `active`), гости с той же расой, но **не** входящие в эту планету (например, только что заполнили анкету и попали в очередь), считаются отдельно.
- Если таких «новых» по этой расе набирается снова **≥ N** (порог со слайдера) — создаётся **вторая планета** (новая группа) для этой же расы с новым заданием. Старая планета продолжает идти со своим таймером и заданием.
- Итого: у одной расы могут быть несколько активных планет (несколько кружков/блоков на экране: «Ворлоки — группа 1 (задание, таймер)», «Ворлоки — группа 2 (задание, таймер)» или визуально различать по названию планеты).

---

## 7. Завершение задания: фото

- Задание считается выполненным только когда к планете **прикреплено фото**. Планета исчезает с экрана только в этот момент.
- **Несколько активных планет на экране**: каждая отображается; по **клику на планету** открывается **модальное окно** с текстом задания, таймером (или «просрочено»), **загрузкой/съёмкой фото** и кнопкой «Завершить задание».
- **Кто прикрепляет фото**: (1) оператор с админки штаба заданий — открывает планету, модалка, фото; (2) можно открыть **ссылку на режим штаба с телефона** (публичная или с кодом), выбрать свою планету в списке, открыть ту же модалку и приложить фото (участники могут завершить задание сами).
- **Фото с таймером 3 секунды**: перед съёмкой обратный отсчёт 3, 2, 1 — затем снимок или загрузка файла. Фото в Supabase Storage, URL в БД (`planets.photo_url` или `planet_tasks.photo_url`). После прикрепления — планета в статус `completed`.

---

## 8. Визуал и UX (кратко)

- Кружки рас — по кругу; внутри — аватарки (или «?») и счётчик участников.
- Крупный шрифт для текста задания при активной планете.
- Таймер 5 минут — хорошо читаемый (крупно).
- Фото: экран/модалка с обратным отсчётом 3 сек и затем кнопка съёмки/загрузки.

---

## 9. Уточнения (принято)

- **Участники кружка** — только те, кто в очереди на кофе (pending/ready), с этой расой; при готовности кофе не исчезают; исчезают только когда планета завершена (фото приложено).
- **Планета не исчезает по таймеру** — только пометка «просрочено», фото можно приложить в любой момент.
- **На экране** — несколько активных планет; на каждую можно нажать → модалка с загрузкой фото и завершением.
- **Фото** — оператор с админки или участник по ссылке режима штаба с телефона: открыть ссылку → выбрать свою планету → модалка → фото → завершить.

## 10. Название и обоснование (космо-фантастика)

Режим нужно назвать и обосновать в сеттинге: **секретное задание для жителей планеты**, которое они должны выполнить.

**Варианты названия режима:**

| Название | Обоснование в сеттинге |
|----------|------------------------|
| **«Миссия „Печать“»** | Центральное командование выдаёт каждой планете запечатанную миссию; вскрыть и выполнить до истечения срока. |
| **«Директива сектора»** | Секретная директива для экипажей по расовой принадлежности: доказать боевую сплочённость фотоотчётом. |
| **«Код контингента»** | По коду контингента (раса) группа получает одноразовое задание; не выполнить — не пройти допуск в следующий сектор. |
| **«Операция „Кольцо“»** | Задание из Кольца (общая база); каждая раса выполняет свой фрагмент операции, фото — доказательство выполнения. |
| **«Брифинг биометрии»** | Система по биометрии (результат квиза) назначает контингенту секретную миссию; выполнить и зафиксировать — обязанность жителей планеты. |
| **«Чёрный ящик планеты»** | Каждой «планете» (группе расы) выдаётся чёрный ящик с заданием; вскрыть, выполнить, запечатлеть. |

**Короткие слоганы для экрана:**
- *«Секретная миссия для вашей расы. Выполните до истечения срока.»*
- *«Директива вскрыта. Жители планеты — к выполнению.»*
- *«Задание контингента. Фотоотчёт обязателен.»*

Можно взять один вариант или скомбинировать (например, название «Миссия „Печать“», слоган про директиву). Если нравится направление — можно сузить до 1–2 вариантов и вшить в интерфейс (заголовок режима, подпись под кругами, текст в модалке).

## 11. План реализации (черновик)

1. **Настройки**: слайдер 3–10, сохранение в localStorage (ключ вида `tamada_min_players` — внутренний, без переименования).
2. **API/данные**:
   - Список рас + по каждой расе гости из очереди: заказы со статусом **pending или ready** (не только pending), группировка по расе. Гости в активных планетах не входят в «свободный пул» для кружка и для новой группы.
   - Публичная (или с коротким кодом) **ссылка на режим тамады** для участников: открыть с телефона → список активных планет → выбрать свою → модалка с фото и завершением.
3. **Экран кругов**: компонент с расами по окружности, внутри кружка — аватарки + счётчик; общее число участников.
4. **Авто-запуск**: при изменении очереди проверять для каждой расы: количество «свободных» участников (не в активной планете) ≥ N → создать планету, выдать задание, запустить таймер 5 мин.
5. **Модель «несколько планет»**: создавать новые планеты по расе при новом наборе N человек; на экране отображать все активные планеты (или список с таймерами).
6. **БД**: поле для фото у планеты или у `planet_tasks` (например `photo_url`).
7. **UI фото**: экран/модалка с таймером 3 сек и вызов камеры/загрузка файла, сохранение в Storage и запись URL в БД, переход планеты в `completed`.
